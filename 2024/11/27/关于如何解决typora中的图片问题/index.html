关于如何解决typora中的图片问题

前言

相信各位同学如果有接触过用markdown文件写笔记的话，一定会被其图片存储问题气到吐血过。经常出现把markdown文件的位置换一下结果里面的图片全部消失不见的情况。如果你设置了把图片单独拿个文件夹存储的话还好，至少把文件中的图片路径改完还能显示出来，要是找不到或者是记不得图片放哪里的话基本上这篇markdown笔记就废了，只能删除重写。这种情况作者本人遇到过也是气的不轻，几次想换其他笔记软件做笔记，奈何typora写笔记风格太好看了，于是就去网上找到了解决方法——图床

常见的问题

有的同学平时用markdown文件写博客，想上传到github或者csdn又或者博客园上，结果发现明明在本地上图片还有的，结果发现上传以后图片全不见了就剩下一个路径

原因

  1. Markdown文件上传到Github，或者把含有图片的md文件传给其他人。
  2. Typora写的md文件中的图片都是另外存储的，不像word软件是存放到word文件里的。

在typora中显示图片方法的优缺点

方案一存储到本地（不适合小编写文章）

优点：

  • 配置简单，免费

缺点：

  • 文件分享麻烦，上传到博客导致图片找不到等问题。

设置如下，每次传md文件给其他人的时候，需要把图片文件夹也一并压缩转发才行，记住勾选“优先使用相对路径”

方案二：Tyora+PicGo+SM.MS（实现免费图片服务器）

优点：

  • 免费
  • 搭建图床并自动上传（适合小编写文章）

缺点：

  • 配置复杂
  • 查看文件需要联网（图片全都是放到SM.MS上的）
  • 文件中的图片加载速度，取决于本机的网速
  • Tyora和PicGo这项功能是2020年最近才有的，还存在不少的问题。

图床的概念

图床，顾名思义，就是存放图片的云端仓库。这种在线图片存储服务提供了稳定、快速的图片托管服务，当我们把本地图片上传至图床后，会获得一个唯一的URL链接，将其插入Markdown文档，即可在任何环境下保证图片的正常加载与展示，极大地提高了文档分享和发布的便利性。
利用图床，不仅能解决图片在网络传输中的速度问题，还能有效节省本地存储空间，尤其对于需要频繁更新和分享的文档，更是不可或缺的工具。
图床详细概念请参考百度百科的解释

解决方法

1.安装PicGo

下载地址我给出来了，Releases · Molunerfinn/PicGo (github.com)

在上github的时候最好挂个代理（上github的话steam++完全足够用了，且免费）

连接Typora与PicGo

  • 打开Typora，进入“文件” -> “偏好设置” -> “图像”选项卡；快捷键: Ctrl + 逗号
  • 在“图像上传服务”下拉菜单中选择“PicGo(app)”；
  • 点击“选择…”，定位并添加PicGo.exe应用程序路径，保存设置后即可启用图床功能

这里图床不一定要选择SM.MS，其他很多像阿里云，github都是可以的，只是其他的不是要钱就是配置麻烦，筛选以后还是感觉这个SM.MS图传最好用（我没有打广告）

注册SM.MS账号,获取access token

  • 点击打开SM.MS]网站，网址https://sm.ms/home/apitoken，注册登录后点击 “API Token” ，复制Token。

配置PicGo

  • 打开安装在本地的PicGo.exe软件，点击“图床设置” -> “SM.MS” ,将刚才网站的Token复制进”设定Token“中。
    • PicGo会在每次需要上传的时候，在后台上传到图床上，这一步几乎代表完成了
  2. PS: Typoa设置里面的上传图片验证失败也没关系，插入一张图片，提示上传成功或者图片连接不是本地而是http开头就代表成功

四、Typora中图片上传与插入的实际操作
单张图片上传

拖拽图片至Typora编辑区，或使用快捷键Ctrl+V粘贴图片，Typora会触发PicGo进行图片上传；
上传成功后，Typora自动将图片转换为Markdown格式链接并插入文档，例如：。

多张图片上传

这边需要下载picgo-plugin-pic-migrater这个插件

这个插件能够实现markdown文件中多张图片的同时上传，这里可能是我picgo的版本问题，下载插件的时候老是显示错误，这边可以点击右上角的下载图标导入本地插件

PicGo/picgo-plugin-pic-migrater: :truck:A PicGo plugin for pictures in markdown files migrating from one picBed to another one. (github.com)

给出了插件的下载地址，只要下载下来解压以后再导入即可正常使用（亲测喵）
